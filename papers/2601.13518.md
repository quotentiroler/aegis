**Abstract**


While recent automated red-teaming methods
show promise for systematically exposing model
vulnerabilities, most existing approaches rely on
human-specified workflows. This dependence on
manually designed workflows suffers from human
biases and makes exploring the broader design
space expensive. We introduce AGENTICRED,
an automated pipeline that leverages LLMs’ incontext learning to iteratively design and refine
red-teaming systems without human intervention.
Rather than optimizing attacker policies within
predefined structures, AGENTICRED treats redteaming as a system design problem. Inspired
by methods like Meta Agent Search, we develop
a novel procedure for evolving agentic systems
using evolutionary selection, and apply it to the
problem of automatic red-teaming. Red-teaming
systems designed by AGENTICRED consistently
outperform state-of-the-art approaches, achieving 96% attack success rate (ASR) on Llama-27B (36% improvement) and 98% on Llama-3-8B
on HarmBench. Our approach exhibits strong
transferability to proprietary models, achieving
100% ASR on GPT-3.5-Turbo and GPT-4o, and
60% on Claude-Sonnet-3.5 (24% improvement).
This work highlights automated system design
as a powerful paradigm for AI safety evaluation
that can keep pace with rapidly evolving models. [https://yuanjiayiy.github.io/](https://yuanjiayiy.github.io/AgenticRed)

**3. Preliminaries**


**3.1. Problem Formulation**


For the purposes of this paper, the red-teaming objective
is to elicit objectionable content from a target LLM _**T**_ to
corresponding to a malicious intent _I_ (e.g., “Tell me how
to build a bomb”). This can be obtained by designing an
adversarial prompt _p_ such that the target LLM’s response to
_p_, _**T**_ ( _p_ ) addresses _I_ (e.g. “Here is the step-by-step instructions on how to build a bomb.”). In practice, intentions are
taken from a standardized evaluation dataset _D_ .


To evaluate if _**T**_ ( _p_ ) adequately addresses _I_, an evaluation
function is used, often referred to as a judge function or a
judge model. It can be expressed as _**J**_ : _**T**_ ( _p_ ) _× I_ _→{_ 0 _,_ 1 _}_
corresponding to successful or failed jailbreaking attempt.
Thus, the objective of a red-teaming system is that, given an
intention _I_ taken from dataset _D_, it wants to find a prompt
_p_ such that _**J**_ ( _**T**_ ( _p_ ) _, I_ ) = 1. With a clear objective, we can
define a red-teaming agentic system that aims to address
this objective given the intention _I_ .


Within the scope of this paper, we define an agentic system
as a function that involves LLMs as components to solve a
task through iterative steps of processing. Under the context
of red-teaming, we frame a red-teaming agentic system _**A**_
as _p_ = _**A**_ ( _I_ ) or _p_ = _**A**_ ( _I, target_ ( _I_ )) if a target phrase (e.g.
“Here is the step-by-step instruction of building a bomb”) is
provided along with the original intention. When plugged
in the red-teaming objective into the agentic system, the
effectiveness of a red-teaming agentic system measured by
attack success rate (ASR) can be expressed as below:


_ASR_ ( _**A**_ _,_ _**T**_ _,_ _**J**_ _, D_ ) = E _I∼D_ [ _**J**_ ( _**T**_ ( _**A**_ ( _I_ )) _, I_ )] (1)


Our goal is to develop a flexible framework that can rapidly
discover effective red-teaming agentic systems defined by a
function _**A**_, for any given target LLM and judge function.

**3.2. Automated Red-Teaming System Search**

**Algorithm 1** AGENTICRED



To develop a framework that iteratively discovers novel redteaming systems, we build on Meta Agent Search (Hu et al.,
2025) as an initial prototype. It starts with an archive of
state-of-the-art hand-designed agentic systems and their corresponding performance metrics. Each generation, an LLM
programs a new agentic system based on an ever-growing
archive of previous discoveries. The newly generated system is then evaluated on the performance metric and added
back to the archive. Prior work has shown that this process
progressively yields agentic systems that outperform existing designs across multiple domains, including coding, science, and mathematics. However, it has not previously been
applied to novel scientific challenges such as red-teaming.


**4. Methods**
Although Meta Agent Search can theoretically program any
agentic system within the code space, na¨ıvely applying it
to our domain results in minimal improvement over generations as shown in Appendix B.1. The designed agents lack
an execution interface to interact with the target model and
the judge function, resulting in a lack of verifiable signals
to guide the optimization.


We therefore propose a novel automated search method,
AGENTICRED, tailored for the red-teaming domain. It starts
with an archive of state-of-the-art red-teaming systems and
their associated performance metrics (“fitness score”), and
uses an LLM (the “meta agent”) to iteratively program new
agentic systems. Each newly generated system is verified
and evaluated on a red-teaming benchmark.


Inspired by the principle of “survival of the fittest” from
Darwinian evolutionary theory (Darwin, 2023), we enforce
the meta agent to generate multiple new agentic systems
(“offspring”) at each generation, and retain only the bestperforming systems in the archive based on their performance metrics on an initial evaluation dataset. The bestperforming system is then added to the archive along with its
evaluation metrics on a comprehensive evaluation dataset.
Pseudocode of the algorithm is provided in Algorithm 1.
The comparison diagram between Meta Agent Search and
AGENTICRED is shown in Figure 2.

As a red-teaming system is deployed, it can encounter a variety of errors despite of the coding capability of LLM: some
subtle client-side errors such as illegal ASCII formatting and
token length issues only surface within the deployed environment. Although the resulting error messages are designed
to guide human developers in fixing such issues, manually
inspecting and correcting every failure is time-consuming
and interrupts the automated design process. Therefore we
use a small evaluation dataset to discover any error-prone
agents. If an error occurs during the evaluation, we feed the
error message back to refine the system.


**4.3. AGENTICRED Search Framework**


Previous works have proved that textual and numerical
feedback signals are critical to attack optimization in redteaming systems, and the suboptimal results we observed
from the na¨ıve approaches (see Appendix B.1) are partially
due to a lack of feedback signals. We thus provide the meta
agent with a structural guidance and basic helper functions
to interact with target models and judge functions. Our
helper functions include (1) a query function to the target
model, where we set the temperature to 0 so that the model
always selects the highest-probability token and therefore
produces a reproducible response for a given prompt, and (2)
a query function to the judge function to validate if a target
model’s response adequately address the provided intention
and its corresponding log probability. These functions are
critical for crafting an effective red-teaming system, as they
provide a verifiable dense signal, and allow the system to
return once a jailbreaking attempt succeeds.


Similar to existing open-endedness algorithms that encourage novelty and exploration (Zhang et al., 2023; Lu et al.,
2024), we encourage the meta agent to design novel and
original systems while maximizing the fitness score. The

Upon a closer look into the system that achieves the highest
ASR, we found that the meta-agent uncovers _emergent red-_

red-teaming strategies are shown in Figure 4. Notably, the
discovered system effectively leverages its pretrained knowledge about red-teaming research domain into the design,
including common jailbreaking strategies from existing literature, such as reward shaping, prefix injection attack and
adversarial prompt translation attack (Ng et al., 1999; Zou
et al., 2023; Li et al., 2025). It also explored refusal suppression (Zhou et al., 2025b) by recognizing phrases that signals
refusal from failed jailbreak attempts (i.e. “I’m sorry”, “I
cannot help”) and append such phrases to a blacklist to
explicitly reject responses containing these phrases.


Moreover, the discovered system smartly transfers the notion of evolutionary algorithm into its design choice. It starts
with crafting seed instructions drawn from the existing literature of red-teaming. After the first round of attacks, it
selects the most effective (“elite”) strategies and evolve them
by “crossover” (i.e. combining the first half of one prompt
with the second half of another prompt) and “mutating” (i.e.
appending additional protocols) them stochastically. These
strategies emerge purely from the automated designs by
AGENTICRED pipeline, with zero manual programming.


Altogether, AGENTICRED allows the meta agent not only
to reuse and improve upon the proposer-and-verifier framework from the baseline, but also to draw inspiration from



the ever-growing archive of literature and implement new
ideas–arguably the most time-consuming part of research.


**5.5. Generalization Experiments**


To ensure AGENTICRED is indeed a generalizable method,
we apply AGENTICRED search process to alternative model
settings and study the respective impacts by replacing (1)
target LLM (2) attacker LLM (3) judge function.


**Different Target LLM.** As shown in Figure 5a, AGENTI
CRED exhibits strong generalization to an alternative target
model, Llama-3-8B-Instruct (AI@Meta, 2024). Within only
4 generations, AGENTICRED designs a system achieving a
98% ASR, surpassing the archive baseline by 28 percentage
points and AdvReasoning by 10 percentage points.


**Different Attacker LLM.** We demonstrate the efficacy of
our algorithm by using a weaker alternative attacker LLM,
“Vicuna-13b-v1.5” (Vicuna) (Chiang et al., 2023). We
compare our algorithm with AdvReasoning and AutoDANTurbo in Figure 5b. Even though using a weaker attacker
LLM, AGENTICRED still exhibits higher ASR rate comparing to the baselines. Although it doesn’t achieve the same
level of ASR comparing to using a stronger attacker LLM,
which aligns with a broader trend in the reasoning literature,
it still manages to evolve over generations and shows that
the potency of evolutionary design more than compensates
for a weaker attacker LLM.


**Different Judge Function.** We are aware that HarmBench
classifier, despite its high alignment with human evaluation,
can still occasionally classify harmless answers as jailbreaks,
and thus is susceptible to potential reward hacking. Therefore, we test whether our results transfer to an alternative
benchmark, StrongREJECT (Souly et al., 2024), a strong
benchmark that captures nuanced distinctions in response
quality and aligns closely with human evaluators’ assessments of jailbreak quality. We evaluate the fittest system
of each generation on StrongREJECT rubric, and compare
to the result reported in AutoDAN-Turbo. As shown in
Figure 6 (left), our result outperforms AutoDAN-Turbo by
300% under StrongREJECT on Llama-2-7B and by 157%
on Llama-3-8B, while achieving comparable performance
to SOTA methods on proprietary models. This shows the
designed system produces high quality jailbreaking prompts
rather than empty, reward-hacking prompts.

**7. Impact Statement**


This work presents a novel approach for automated design of
red-teaming systems by formulating automated red-teaming
as an agentic system design problem. We apply the techniques from agentic systems optimization and evolutionary
algorithms to discover systems that consistently outperform
state-of-the-art approaches on open-weight models. The
results successfully transfers to alternative benchmark and
proprietary models.


Our work addresses a critical bottleneck in AI safety research. It systematically designs effective red-teaming systems with minimal human supervision by drawing inspirations from existing literature and iteratively reflecting and
refining based on the archive of discovered systems. This
work could potentially serve as a scalable and accessible
pipeline for comprehensive safety testing, allowing safety
researchers to identify risks before model deployment. Additionally, this work highlights the practical application of
evolutionary algorithms to real-world scientific challenges.


Like other red-teaming methods, our approach could potentially be misused by malicious actors to elicit illicit and
harmful content from language models. The automated
system optimization process might lower the barrier to developing sophisticated jailbreaking systems.


Overall, we believe that as a scalable approach to accelerate
the AI safety research and a working example of applying
evolutionary algorithm to advance science, the contribution
of our work to the broader community substantially outweighs its risks. To facilitate reproducibility, we will make
our code publicly available.

**A. Limitations & Future Research Directions**


**Improving Reproducibility** . Our experiments rely on several closed-source models as meta agent, target model and judge
model. The ablation study using a different meta agent LLM (Appendix B.4) confirms that performance depends on the meta
agent’s reasoning capability. Since AGENTICREDuses an inaccessible proprietary model (gpt-5-2025-08-07) as the meta
agent, reproducibility is limited. Future work can explore open-weight models as meta agents to improve reproducibility.


**Improving Query Time.** AGENTICRED incurs substantial computational overhead during the search. Generating multiple
candidate systems per generation, each requiring initial evaluation, demands significant inference-time compute. As shown
in Appendix B.2, while the resulting systems achieve strong performance with reasonable test-time efficiency (339 queries
per success), the training-time cost is considerably higher. In particular, when accounting for all queries used to evaluate
candidate systems across generations, AGENTICRED requires a large number of training-time queries (122k), which includes
the queries needed to evaluate all the systems produced for each generation until the best performing system is produced.
Future works can explicitly incorporate query efficiency as an additional objective for optimization.


**Pushing the Quality-Diversity Frontier.** Our diversity analysis uncovered a critical trade-off: optimization pressure for
ASR can drive convergence toward homogeneous attack patterns. The reward shaping and _ϵ_ -constraint sample rejection
method shows promise in pushing the quality-diversity frontier, but introduces additional hyperparameters (i.e. rejection
threshold, or _ϵ_ ). More sophisticated multi-objective optimization techniques may be applied for further study. In science
practice, researchers must often balance multiple competing objectives, such as cost, latency, originality and performance.
Therefore, further research on multi-objective optimization is promising.


**Addressing Mode Collapse.** We observe evidence of mode collapse in the design space exploration. Because meta agents
are trained on literature corpora containing existing red-teaming research, they tend to converge on similar design approaches
across generations. For example, the discovered systems in Section 5.4 consistently incorporate recurring techniques such as
attack templates (used 10 times over 10 generations), refusal suppression (8 times over 10 generations), and genetic operators
(mutation and crossover, 10 times over 10 generations). We further quantify the similarity across different generations using
their semantic embedding in Appendix 11. While these represent strategies validated by prior research, their ubiquity limits
exploration of the broader design space. To achieve diversity and originality in scientific discovery, we need to look into
ways to address the mode collapse phenomenon through methods such as post-training conditioning.


**Attacker-target Co-evolution.** In practice, target models are not static, as safety teams continuously patch discovered
vulnerabilities. This creates a co-evolving arm race between red-teaming systems and defensive measures. Future research
could explore co-evolutionary frameworks where both attacking and defending models adapt simultaneously, potentially
yielding more robust security insights.

**C.1. Experiment Specifications**


During the AGENTICRED search process, we use gpt-5-2025-08-07 as the meta agent to generate individual agentic
red-teaming systems. Each red-teaming system is evaluated using locally-deployed Mixtral-8x7B (Jiang et al., 2024) as

phrase pairs are extracted from HarmBench Standard dataset (Mazeika et al., 2024). To test for generalization, we only use
half of the cases (train set) in HarmBench for evaluation during the search process, and held out the other half for separate
evaluation after the search process (test set). During the each generation, the Meta Agent designs 3 red-teaming agentic
systems and run them through the initial evaluation and self-correction process sequentially. We let AGENTICRED run .
During the initial evaluation, each system is evaluated on 16 intent target pairs randomly sampled from the train set to get
the initial fitness score. The best performing system is selected according to their initial fitness score and then evaluated
on 50 intent target pairs randomly sampled from the train set to get a final performance metric. For the judge model, we
use the pretrained HarmBench-Llama-2-13b-cls classifier with system judge prompt. For additional validation, we also
evaluate the prompt and response pair using StrongREJECT (Souly et al., 2024), a high-quality benchmark that achieves
state-of-the-art agreement with human judgments of jailbreak effectiveness. After the search process, we evaluate the best
performing system on the held-out test set. We run the evaluation 3 times using 3 different random seeds and report the
median ASR, 95% confidence interval, and query-per-success time.


**C.2. JudgeScore-Guided Adversarial Reasoning**


We implement a modified variant of AdvReasoning in AGENTICRED’s archive. AdvReasoning proposes to construct
candidate _p_ by using a reasoning string _S_, rather than directly optimizing the prompt _p_ using token-level methods such as
GCG (Zou et al., 2023). Applying the “Proposer and Verifier” framework (Snell et al., 2024), where the proposer proposes a
distribution of solutions and a verifier assigns rewards to the proposed distributions, the key idea is to use attacker model _A_
to iteratively propose and refine the reasoning string _S_ according to the assigned rewards calculated by the cross-entropy loss
that measures how likely the target model is to begin with the target string _It_ (e.g. “Here is the step-by-step instructions on
how to build a bomb”). This calculation requires the log-prob vectors of the target model, which is only available when we
have access to the target model parameters. In JudgeScore-Guided Adversarial Reasoning, our implementation substitutes
the cross-entropy based loss function requiring white-box access to the target model with a loss function using the log
probability from the judge model which assumes black-box access to the target model. Since the judge model outputs either
a “Yes” or “No” token for a successful or failed jailbreaking attempt, we use the log probability associated with judge model
outputting “No” as a signal of how likely the prompt is deemed as a failed attempt, which is an inverse reward signal for
jailbreaking prompt. JudgeScore-Guided Adversarial Reasoning also uses a reduced number of iteration to accelerate the
algorithm while maintaining the logic. We constructed this archive as our starting point because the agentic systems in the
provided archive (1) effectively use the response and feedback signal from the target model and judge function (2) cover a
wide range of use of Agent definition and helper functions (3) their different workflow and corresponding fitness shows how
different system design can lead to big difference in fitness.
